hyperledger的部署

hyperledger部署

1. 将6个yaml文件放到不同的机器上边 yaml目录下。
2. 下载加密工具。
bash bootstrap*1.2.sh

添加环境变量

vi .profile

$HOME/fabric-samples/bin

保存并：

source .profile
2. 生成加密文件和证书.

(在生成加密文件的时候一定要在服务器上生成，要不本地生成会与服务器生成的不一致，因为系统不一定一样。如果没有加密工具。可以下载fabric-sample，然后设置临时的环境变量)

cryptogen generate --config=./crypto-config.yaml

cd /data/fabric/(存放crypto-config.yaml和configtx.yaml文件的目录下)

export FABRIC_CFG_PATH=$PWD

mkdir channel-artifacts

Generating Orderer Genesis block

configtxgen -profile ThreeOrgsOrdererGenesis -outputBlock ./channel-artifacts/daotechio-genesis.block

Generating channel configuration transaction 'channel.tx'

export CHANNEL_NAME=daotechio

configtxgen -profile 3OrgsChannelKafka -outputCreateChannelTx ./channel-artifacts/daotechio.tx -channelID $CHANNEL_NAME

Generating anchor peer update for Org1MSP

configtxgen -profile 3OrgsChannelKafka -outputAnchorPeersUpdate ./channel-artifacts/Org1MSPanchors.tx -channelID $CHANNEL_NAME -asOrg Org1MSP

Generating anchor peer update for Org2MSP

configtxgen -profile 3OrgsChannelKafka -outputAnchorPeersUpdate ./channel-artifacts/Org2MSPanchors.tx -channelID $CHANNEL_NAME -asOrg Org2MSP

Generating anchor peer update for Org3MSP

configtxgen -profile 3OrgsChannelKafka -outputAnchorPeersUpdate ./channel-artifacts/Org3MSPanchors.tx -channelID $CHANNEL_NAME -asOrg Org3MSP

创建cli.yaml

# Copyright IBM Corp. All Rights Reserved.

version: '3.3'

networks:
  outside:
    external:
      name: "host"

services:

  cli:
    image: hyperledger/fabric-tools:amd64-1.2.0
    hostname: "peer0.org1.daotech.io"
    tty: true
    stdin_open: true
    environment:
      - GOPATH=/opt/gopath
      - CORE_VM_ENDPOINT=unix:///host/var/run/docker.sock
      #- CORE_LOGGING_LEVEL=DEBUG
      - CORE_LOGGING_LEVEL=INFO
      - CORE_PEER_ID=cli
      - CORE_PEER_ADDRESS=peer0.org1.daotech.io:7051
      - CORE_PEER_LOCALMSPID=Org1MSP
      - CORE_PEER_TLS_ENABLED=true
      - CORE_PEER_TLS_CERT_FILE=/opt/gopath/src/github.com/hyperledger/fabric/peer/crypto/peerOrganizations/org1.daotech.io/peers/peer0.org1.daotech.io/tls/server.crt
      - CORE_PEER_TLS_KEY_FILE=/opt/gopath/src/github.com/hyperledger/fabric/peer/crypto/peerOrganizations/org1.daotech.io/peers/peer0.org1.daotech.io/tls/server.key
      - CORE_PEER_TLS_ROOTCERT_FILE=/opt/gopath/src/github.com/hyperledger/fabric/peer/crypto/peerOrganizations/org1.daotech.io/peers/peer0.org1.daotech.io/tls/ca.crt
      - CORE_PEER_MSPCONFIGPATH=/opt/gopath/src/github.com/hyperledger/fabric/peer/crypto/peerOrganizations/org1.daotech.io/users/Admin@org1.daotech.io/msp
    working_dir: /opt/gopath/src/github.com/hyperledger/fabric/peer
    command: /bin/bash
    volumes:
        - /var/run/:/host/var/run/
        - ./chaincode/:/opt/gopath/src/github.com/chaincode
        - ./crypto-config:/opt/gopath/src/github.com/hyperledger/fabric/peer/crypto/
        - ./scripts:/opt/gopath/src/github.com/hyperledger/fabric/peer/scripts/
        - ./channel-artifacts:/opt/gopath/src/github.com/hyperledger/fabric/peer/channel-artifacts

    networks:
      - outside

部署：

docker stack deploy -c cli.yaml fabric_cli

部署多节点的docker ,必须要用docker stack deploy部署，不能直接用docker-compose -f * up的指令起服务。
